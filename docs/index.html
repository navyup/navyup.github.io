一、准备主机环境

  • 操作系统：CentOS 7

  • 软件：Docker 最新稳定版，详见docker安装配置手册

  • 网络：配置IP, NETMASK, GATEWAY, DNS, 等

    • IPADDR = 192.168.222.23
    • NETMASK = 255.255.255.0
    • GATEWAY = 192.168.222.1
    • DNS1 = 114.114.114.114

  • 主机名：Rancher

  • 防火墙：关闭状态且禁止开机自启

  • 时钟服务：在服务端开始时钟服务，保证集群时钟一致，详见附件

 

二、Rancher Server 安装与配置

 

1. 安装Rancher server

在已安装的docker的CentOS 7 上 直接执行命令：

​x
root@Rancher#docker run -d --restart=unless-stopped -p 192.168.222.23:80:80 -p 192.168.222.23:443:443  rancher/rancher:stable
​

执行完命令后，docker 检查本地没有rancher镜像便会去网络上拉去镜像，由于国际链路网络速度有限，建议配置阿里加速器，加速镜像下载。具体如下：

xxxxxxxxxx
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": ["https://ojc86f6u.mirror.aliyuncs.com"]
}
EOF
sudo systemctl daemon-reload
sudo systemctl restart docker

其中ojc86f6u是我本人的阿里加速器的地址。

四、总结

第一次使用Ranher Server 2.x, 踩了许多坑，在此予以总结，免得他人少走弯路。使用Rancher Server 2.x 需要对Docker 和Kubernetes有所了解，特别相关组件和概念，这样会帮助用户明白各个Rancher Server节点角色与Kubernetes各个节点以及组件的对应关系，会帮助用户理解什么角色的组件要高可用，便可以顺利打造或者建造高可用集群。

 

另外，Rancher拥有多个产品，如Rancher Kubernetes Engine(RKE) 是一个Kubernetes公开发行版，可以帮助用户快速建立Kubernetes集群，因此建议用户对Rancher的产品体系和发展历史有一定的了解。值得注意的是Rancher Server 2.x 和Rancher Server 1.x完全是两个不同内涵和外延的版本，在1.x中部分组件或者叫功能已经不再支持和维护。2.x是为Kubernetes生态而打造的100%开源的容器管理平台，并支持主流的公有云平台。利用Rancher Server 2.x可以轻松的构建Kubernetes集群，也可以纳管已有的Kubernetes集群。

 

TIPS

以下是安装过程踩过的雷，总结在此：

    1. 集群时钟须有统一的基准，可以使用Rancher Server 作为时钟服务器，其他节点作为客户机从时钟服务器同步时间。在节点注册过程中发现，有的节点注册一直注册出错，且不好查找，后经统一时钟后，节点注册正常。
    1. 初学者建议直接关闭防火墙，或者开启必要的相关端口。
    1. 由于Rancher软件资源服务器都在国外，建议配置阿里云加速器。详见Q&A
    1. Rancher节点再向Rancher Server 2.x 注册时，一旦出错或者Rancher Server 更换主机或者IP时，一定要彻底清理环境再重新注册，具体清理环境的办法可执行clear.sh脚本，详见附件。

2. 2_clear.sh

xxxxxxxxxx
#!/bin/bash
#******************************************************************
# Author: Rancher Labs.
# Date: 2020.05.29
# Last modified: by Haijun Ren on 2020.06.03
# ******************************************************************
​
sudo yum install -y net-tools
​
KUBE_SVC='
kubelet
kube-scheduler
kube-proxy
kube-controller-manager
kube-apiserver
'
​
for kube_svc in ${KUBE_SVC};
do
  # 停止服务
  if [[ `systemctl is-active ${kube_svc}` == 'active' ]]; then
    systemctl stop ${kube_svc}
  fi
  # 禁止服务开机启动
  if [[ `systemctl is-enabled ${kube_svc}` == 'enabled' ]]; then
    systemctl disable ${kube_svc}
  fi
done
​
# 停止所有容器
docker stop $(docker ps -aq)
​
# 删除所有容器
docker rm -f $(docker ps -qa)
​
# 删除所有容器卷
docker volume rm $(docker volume ls -q)
​
# 卸载mount目录
for mount in $(mount | grep tmpfs | grep '/var/lib/kubelet' | awk '{ print $3 }') /var/lib/kubelet /var/lib/rancher;
do
  umount $mount;
done
​
# 备份目录
mv /etc/kubernetes /etc/kubernetes-bak-$(date +"%Y%m%d%H%M")
mv /var/lib/etcd /var/lib/etcd-bak-$(date +"%Y%m%d%H%M")
mv /var/lib/rancher /var/lib/rancher-bak-$(date +"%Y%m%d%H%M")
mv /opt/rke /opt/rke-bak-$(date +"%Y%m%d%H%M")
​
# 删除残留路径
rm -rf /etc/ceph \

    /opt/cni \

    /run/calico \

    /var/run/calico
​
# 清理网络接口

4. 集群时钟统一

需要在集群每个节点上安装ntp和ntpdate 两个软件，并以Rancher Server 作为集群的时间服务器，其他节点从时间服务器上同步时间。

主要的配置文件是/etc/ntp.conf , 服务器和节点的配置方法略有不同。

时间服务器配置如下：

xxxxxxxxxx
# For more information about this file, see the man pages
# ntp.conf(5), ntp_acc(5), ntp_auth(5), ntp_clock(5), ntp_misc(5), ntp_mon(5).
​
driftfile /var/lib/ntp/drift
​
# Permit time synchronization with our time source, but do not
# permit the source to query or modify the service on this system.
restrict default nomodify notrap nopeer noquery
​
# Permit all access over the loopback interface.  This could
# be tightened as well, but to do so would effect some of
# the administrative functions.
restrict 127.0.0.1 
restrict ::1
​
# Hosts on local network are less restricted.
    restrict 192.168.222.0 mask 255.255.255.0 nomodify notrap
​
# Use public servers from the pool.ntp.org project.
# Please consider joining the pool (http://www.pool.ntp.org/join.html).
#server 0.centos.pool.ntp.org iburst
#server 1.centos.pool.ntp.org iburst
#server 2.centos.pool.ntp.org iburst
#server 3.centos.pool.ntp.org iburst
server ntp3.aliyun.com iburst
​
#broadcast 192.168.1.255 autokey    # broadcast server
#broadcastclient            # broadcast client
#broadcast 224.0.1.1 autokey        # multicast server
#multicastclient 224.0.1.1      # multicast client
#manycastserver 239.255.254.254     # manycast server
#manycastclient 239.255.254.254 autokey # manycast client
​
# Enable public key cryptography.
#crypto
​
includefile /etc/ntp/crypto/pw
​
# Key file containing the keys and key identifiers used when operating
# with symmetric key cryptography. 
keys /etc/ntp/keys
​
# Specify the key identifiers which are trusted.
#trustedkey 4 8 42
​
# Specify the key identifier to use with the ntpdc utility.
#requestkey 8
​
# Specify the key identifier to use with the ntpq utility.
#controlkey 8
​
# Enable writing of statistics records.
#statistics clockstats cryptostats loopstats peerstats
​
# Disable the monitoring facility to prevent amplification attacks using ntpdc
# monlist command when default restrict does not include the noquery flag. See
